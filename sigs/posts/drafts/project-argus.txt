## The Plan

### Network Access Control
- RADIUS server (Authentik) for 802.1X wireless authentication
- MAB (MAC Authentication Bypass) for wired port authentication on MikroTik
- Certificate authority for device certificates
- Tighten inter-VLAN firewall rules (replace temporary allow-all rules based on observed traffic patterns)

### Network Visibility
- NetFlow/sFlow collection from MikroTik
- IPFIX export to SIEM for network/host event correlation
- IDS/IPS (Suricata plugin on OPNsense)

### SIEM & Threat Detection
- SIEM (Wazuh)
- Endpoint Detection & Response (Wazuh agents for HIDS and Active Response)
- Deep Kernel Telemetry (Sysmon for Windows / Auditd for Linux integrated into SIEM)
- Open-Source Antivirus Engine (ClamAV integrated with Wazuh FIM for real-time malware scanning)
- Live Digital Forensics & Incident Response (Velociraptor for active threat hunting)

### Deception & Active Defence
- Honeypots on isolated VLANs
- Network-based honeypots — fake servers on unused IPs in each VLAN that should never receive traffic, any connection is an immediate alert
- Active Deception & Tarpits (Endlessh to trap automated scanners)
- DNS Sinkhole Shaming (Redirecting banned traffic to a local "VLAN Violation" landing page)
- Digital Landmines (Deploying Canarytokens in sensitive directories and repos)

### Identity & Access
- Authentik IDP deployment
- SSO across all internal services via OIDC/SAML
- PKI and cert management
    - SSH certificates and local CA for *.local.eddiequinn.casa 

### Monitoring & Alerting
- Vulnerability scanning
- Physical Alert Escalation (Home Assistant webhooks for high-severity event tracking)
- "Quis Custodiet" Guard (Dead Man's Switch monitoring via external heartbeat)
- Thermal Panic Automation (IPMI-driven graceful VM shutdowns during ambient heat spikes)

### Hardening & Compliance
- Incident response runbooks
- OpenSCAP for automated compliance checking against a baseline
- Automated config backups for OPNsense and MikroTik with diff alerting
- Log Apocalypse Mitigation (Logrotate and retention policy execution)
- Tighten DevSecOps pipelines

### Identity & Access — VM Fleet Integration
- VM-level identity via Authentik LDAP (sssd/PAM/nsswitch) — replaces per-VM Ansible user provisioning
- Migration of existing VM fleet to NixOS to support this as shared, declarative config rather than per-host setup
    - Reusable `ldap-auth.nix` module imported by every host — single point of change for LDAP config across fleet
    - Scope boundary: VM login only (SSH, local users/groups, sudo via LDAP group mapping). Container/service-level auth on these VMs remains a separate concern.
- OpenTofu (via `bpg/proxmox` provider) for VM provisioning on Proxmox
    - Pre-baked NixOS images (`nixos-generators`) cloned per VM, rather than live config push to bare VMs
    - Static IPs per VM via existing DHCP reservation scheme (Blackwall-aligned)
- Migration approach: one-by-one cutover per VM (stand up NixOS replacement, verify, decommission original) rather than fleet-wide big-bang
- Open decisions, to resolve at build time:
    - TLS to Authentik LDAP outpost — proper PKI cert (tie-in to PKI work above) vs. interim self-signed/StartTLS
    - Sudo/group mapping model — flat vs. distinct admin/non-admin LDAP groups
- Note: this work was scoped out of Project Blackwall during VM-auth planning; NixOS/OpenTofu chosen as least-effort path for this specific problem, not as an early start on Full IaC project (though it happens to advance it)
