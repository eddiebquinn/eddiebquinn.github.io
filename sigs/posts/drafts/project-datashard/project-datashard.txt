Project Datashard
In the Cyberpunk universe, a datashard is a small piece of physical media — slotted into a reader, cracked open, and consumed. They contain everything: mission briefings, personal logs, corporate secrets, the last words of someone who didn't make it out. They are the fundamental unit of information in a world where data is currency.
The blog is a datashard. Each post is a shard — a preserved record of thinking at a specific point in time, slotted into the reader and left there. Project Datashard is the work of making the reader worthy of the content.
Objectives

Build a custom Hugo theme layer on top of the existing risotto base, following strict shadowing conventions — no upstream files touched, all modifications in root overrides
Implement a retro-futuristic terminal aesthetic: ASCII art header, blinking cursor micro-interactions, inverse video hovers, custom text selection
Build a sidebar system status widget formatted as a telemetry printout
Implement a vanilla JavaScript theme switcher with three terminal colour profiles — Monochrome, Green Phosphor, and Amber CRT — with state persisted via localStorage
All interactive elements in vanilla JavaScript, zero framework dependencies

Constraints

Hugo configuration and root override architecture only
Mobile-first — all terminal effects must degrade gracefully on small screens
No compromise to readability — the aesthetic serves the content, not the other way around

Why now
Blackwall proved the infrastructure. Argus will prove the security posture. Datashard proves that the surface the world sees is as considered as everything running underneath it. A blog that looks like a terminal and reads like a person is the correct aesthetic for someone who builds enterprise networks in a one-bedroom flat and writes about it at 2am.
Status: Planned. Begins after Project Blackwall is shipped and CCNA study is underway.
